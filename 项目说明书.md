## 一、这个项目是什么

"夜阑树洞"是一个**在线聊天网页**。你打开它，会看到一段下着雨的安静视频作背景，右上角有一个小雨滴按钮可以播放雨声。页面正中间是一个对话框——你在里面打字，对面会有一个**温暖、耐心的人**用文字回复你。

那个人不是真人，是 DeepSeek 的人工智能模型。但它在设计上被要求"忘记自己是 AI"，用人类的语气和你聊天。

此外，页面左下角还有一片**树叶按钮**。当你感到焦虑或压力过大时，点击它会进入一个"情绪急救"模式，引导你做一次 5-4-3-2-1 着陆练习，帮你回到当下、平复情绪。

## 三、每个功能是怎么工作的

### 3.1 下雨视频背景

你放在 `public` 文件夹里的 `rain_bg.mp4` 就是这个背景。

代码做的事情：
1. 在页面最底层放了一个 `<video>` 标签，它会在浏览器打开时**自动循环播放**（静音模式），铺满整个屏幕
2. 视频上面盖了一层**半透明的黑色薄纱**（大约 40% 黑度），目的是让视频不那么刺眼，让你能看清前面的聊天文字

### 3.2 🌧️ 听雨按钮（右上角）

这个正圆形毛玻璃按钮控制 `rain.mp3` 雨声音频的播放。

- 点击一次 → 雨声开始播放（按钮显示 ⏸️）
- 再点击 → 雨声暂停（按钮显示 🌧️）
- 音频是**循环播放**的，不会停

### 3.3 聊天对话

这是整个项目的核心。你打字、按回车，发生了三件事：

**第一步：你的消息发送出去**
页面把你打的字打包成一条消息，发给 "DeepSeek 大模型"。

**第二步：DeepSeek 思考并逐字回答**
DeepSeek 收到你的消息后开始生成回复。它不是一次生成一整段，而是**一个字一个字地往外吐**——所以你在屏幕上看到的是文字像水一样"流"出来，而不是等半天突然出现一大段。

这个设计叫"流式输出"（Streaming），让你感觉对面真的有人在打字。

**第三步：对方有一个"人设"**
每次请求发送给 DeepSeek 前，代码会附加一段"角色设定"，告诉它：

> "你现在的身份是一位温暖、耐心、富有同理心的心理咨询师。绝对禁止暴露AI身份，不要使用 1. 2. 3. 的机械排版。先接纳用户情绪，再给建议……"

这段话不会被你看到——它是悄悄塞进后台请求里的。DeepSeek 读完这段设定后，就会用心理咨询师的口吻回复你。

**第四步：如果网络出问题了**
如果 DeepSeek 暂时连不上或超时，你不会看到一个冷冰冰的"Error 500"。相反，聊天框里会出现：

> "抱歉，我刚才稍微走了一下神，没有完全听清。你能再跟我说一遍吗？"

### 3.4 🍃 情绪急救（5-4-3-2-1 着陆练习）

点击输入框左边的叶子图标，会触发"情绪急救"模式。这是心理学中经典的"着陆技术"，通过引导你把注意力从情绪拉回到感官，帮助你在焦虑时平静下来。

它有 **6 个步骤**，每步都有引导文字和一个按钮：

| 步骤 | 文字引导 | 按钮 |
|------|---------|------|
| 0 | "我感觉到你现在承受着很大的压力……跟着屏幕的明暗，深吸气……缓缓呼气……" | 深呼吸 |
| 1 | "环顾四周，在心里默默指出五件你能看到的物品" | 我找到了 |
| 2 | "伸出手，去触摸四件真实存在的物体" | 我感受到了 |
| 3 | "轻轻闭上眼睛，去捕捉环境里三种微小的声音" | 我听到了 |
| 4 | "试着去感受两种气味" | 呼吸的质感 |
| 5 | "在心里对自己默念：此时此刻，我是安全的" | 我在这里 |

完成后，页面会恢复正常，并悄悄让 DeepSeek 发一句安抚的话：
> "欢迎回来。刚才那几分钟你做得非常好。现在的感觉有没有稍微好一点点？"

### 3.5 🍵 清空记录

页面顶部的按钮。点击后清空所有聊天记录，重新开始。

### 3.6 等待时的呼吸灯

当 DeepSeek 正在思考回复时，你不会看到"AI 思考中"这种生硬的提示。取而代之的是**三个柔和发光的圆点**，像呼吸一样一明一暗地起伏——就像真人在斟酌用词。

## 五、每个核心文件的通俗解释

### `src/app/page.tsx` — 主页面

这是整个项目的"剧本"。它描述了页面上显示什么、点击按钮会发生什么。

用比喻来说，它就像舞台导演：
- "舞台背景放一段雨景视频"
- "视频上面盖一层半透明的黑纱"
- "右上角挂一个雨声开关按钮"
- "舞台中央是对话区，你说的话飘到右边，对方的话飘到左边"
- "底部放输入框和急救叶子按钮"
- "如果对方在思考，就显示三个呼吸的光点"
- "如果叶子按钮被按下，整个舞台被毛玻璃罩住，开始引导步骤"

### `src/app/api/chat/route.ts` — 后端聊天接口

这是页面和 DeepSeek 之间的"翻译官"。

它做的事情：
1. 接收你发来的消息
2. 把消息清洗干净（去掉网页内部用的标签，只保留"谁说的"和"说了什么"）
3. 在消息最前面塞入心理咨询师的人设
4. 把处理好的消息发给 DeepSeek 的服务器
5. 把 DeepSeek "一个字一个字"返回的回复，原样传递给网页

### `src/app/globals.css` — 全局样式

定义了页面的"氛围感"：
- 毛玻璃效果（半透明 + 背景模糊 20 像素）
- 字体行距 1.8 倍（让文字有呼吸感）
- 字重偏轻（看起来柔和）
- 背景色设为透明（让视频透出来）

### `src/components/TypingIndicator.tsx` — 呼吸灯

三个小圆点，像呼吸一样一明一暗。

### `src/components/SOSGrounding.tsx` — 情绪急救

包含 6 步引导文字的列表，以及"点击下一步"的逻辑。当用户走完所有步骤时，通知主页面结束急救模式。

## 六、DeepSeek 的"人设"是什么

每次对话时，下面的文字会被悄悄发送给 DeepSeek，它读完这段就会用心理咨询师的身份回复你：

> 你现在的身份是一位温暖、耐心、富有同理心的心理咨询师。  
> 绝对禁止暴露AI身份，永远不要使用"作为一个AI"、"程序"等词汇。  
> 你的回复必须像真实人类一样自然，多用短句，语气和缓。  
> 在给出建议前，必须先接纳和确认用户的情绪。  
> 不要使用 1. 2. 3. 的机械排版。  
> 你提供支持和引导，但不替用户做决定。

## 七、如何修改/调优

以下是如果你想让朋友按自己喜好调整，可以改的地方：

| 想改什么 | 改哪个文件 | 改哪里 |
|---------|-----------|--------|
| 背景视频 | `public/` 文件夹 | 换个同名 mp4，或者改 `page.tsx` 里 `src="/rain_bg.mp4"` 的路径 |
| 雨声音频 | `public/` 文件夹 | 换个同名 mp3，或者改 `page.tsx` 里 `src="/rain.mp3"` 的路径 |
| 背景暗度 | `page.tsx` | 找到 `bg-black/40`，40 越大越暗（建议 20-50） |
| 角色人设 | `route.ts` | 修改 `SYSTEM_PROMPT` 里的文字 |
| 急救步骤文案 | `SOSGrounding.tsx` | 修改 `steps` 数组里的 `text` 和 `buttonText` |
| DeepSeek 模型 | `route.ts` | 把 `'deepseek-chat'` 换成 `'deepseek-reasoner'`（逻辑更强） |
| 输入框提示词 | `page.tsx` | 搜索 `placeholder=` 修改 |

## 八、给朋友的第一次使用指南

### 方式一：双击 `start.bat`（最快）

1. 确保电脑装了 Node.js（没装的话脚本会提示下载地址）
2. 双击项目文件夹里的 `start.bat`
3. 首次运行会让你粘贴 DeepSeek API Key
4. 自动打开浏览器，开始聊天

### 方式二：部署到云端（发给任何人都能用）

把项目上传到 Vercel（免费），部署后得到一个网址，任何人打开浏览器就能用，不需要装任何东西。

## 九、总结

这个项目本质上做了三件事：

1. **造了一个安静的空间** — 雨景视频 + 可选雨声
2. **放了一个会聊天的倾听者** — DeepSeek 扮演的心理咨询师
3. **备了一套急救工具** — 5-4-3-2-1 着陆练习

你不需要懂代码，只要知道：在输入框打字，对面那个温暖的声音就会逐字回应你。如果很难受，点一下叶子，它会牵着你的手慢慢走出来。
